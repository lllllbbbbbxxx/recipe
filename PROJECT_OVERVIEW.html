一、项目概述

“随手做菜”是一款帮用户用现有食材快速决定今天吃什么、并能把菜单分享给朋友点单的微信小程序。定位不是菜谱大全，而是轻量决策 + 库存管理 + 熟人点单工具。

产品名统一为随手做菜（导航标题仍显示“今天吃什么”作为 slogan）。前端为微信原生（WXML/WXSS/JS），后端用微信云开发（云数据库 + 云函数 + 云存储），OCR 走腾讯云。

数据策略：所有写操作本地缓存优先（即时）+ 云端后台同步（并行、静默失败），保证 UI 不卡顿；读时以云端为准回填缓存。

二、页面结构

首页

menu 菜单

  • 分类列 + 菜谱列表（缺料圆点/标签，左划编辑删除）
  • 今日菜单购物车（食材汇总、复制）
  • 今天抽一顿 / 生成菜单
  • 顶部“分享”→ 勾选菜品分享（快照）
  • 菜谱<10 道时“一键导入家常菜”入口
菜谱

add 添加/编辑

  • 封面（上传前压缩 800px+降质量）
  • 菜名*/类型/标签/食材*（必填带 *）
  • 常用调料一键添加；新食材加在最上面
库存

fridge 冰箱

  • 分类折叠（调料默认收起）、左滑删除
  • 非常备可填数量；空冰箱快捷加常备
  • OCR 拍照/截图识别食材（噪声过滤）
决策

draw 抽一顿

  • 库存匹配过滤 + 5:2 加权抽卡
  • 加入今日菜单 / 再抽 / 分享卡片
分享

share 好友选单

  • 按 menuId 读快照（或旧链接 chefId 读全部）
  • 头像昵称一键授权、点菜提交
  • “我的点单”查看/修改/删除自己的单
接单

orders 接单

  • 厨师查看收到的订单（头像+昵称）
  • 一键加入今日菜单 / 标记完成
分享记录

shares 分享记录

  • 厨师回顾历次分享（菜数/时间/菜名）
  • 任选一条一键再分享，内容不变
备份

backup 数据备份

  • 导出/导入文本、云端快照

五、核心逻辑

抽卡 / 匹配（match-utils.js）

食材按重要性分四级：main(第0个)、required(关键)、optional(葱姜蒜等)、seasoning(调料)。模糊包含匹配（草鱼↔鱼），过期食材不算拥有。canDraw=主料+关键料齐；canCook=全齐；almostReady(差点味)=能抽但缺可选/调料。抽卡池：全齐权重 5、缺可选料权重 2，池内等概率随机 → “可做优先，缺主料抽不到”。

OCR 后处理（fridge.js）

去价格/时间/纯符号（ocrIsJunk：不含中文即噪声）、去按钮/店名/广告词（词库）、拆“菜名+重量”（ocrSplitNameAmount）、去括号备注与“约”尾缀、只保留命中食材词库或扩展词的项（ocrIsFood），命中默认勾选。

分享快照模型（share-store.js）

厨师分享前勾选菜品 → onShareAppMessage 必须同步返回，故 prepareShare 同步生成 menuId 存入实例变量 this._shareMenuId，并后台写入 shared_menus 快照，链接带 menuId。好友按 menuId 读快照（公开可读），只看到那一份；旧链接（仅 chefId）回退到 getChefDishes 全量。分享记录页可用同一 menuId 一键再分享。

去重与批量写（stores）

冰箱 normalizeItems 按 name 归一去重（自动合并历史重复）；saveItemsBatch/saveDishesBatch 先同步写本地缓存（秒开）再并行后台 upsert；删除按名字清云端所有同名文档。

埋点与指标（track.js + analytics 云函数）

事件：app_open(含 dishCount/fridgeCount 快照)/app_hide、page_view、first_action(会话首个关键动作)、dish_add_success、dish_import_done、fridge_add(source)、fridge_qty_edit、recommend_view(match)/recommend_accept/recommend_redraw、generate_menu、meal_decided、meal_cooked、menu_share，以及裂变漏斗事件 share_open(mode: snapshot/chef)、order_submit/order_edit/order_delete、order_accept_to_menu/order_mark_done。每条带 appVersion。

analytics 云函数：7 个产品问题指标 + 次日/七日留存、人均周决策次数、裂变漏斗（分享→打开→下单→被分享者建库转化）、近14天按天聚合(UTC+8)；支持 {days: N} 参数圈定 Q1-Q7 统计窗口（留存/漏斗恒为全量）。

九、开发注意事项

WXML 绑定不支持方法调用（.includes()/.map() 等）——选中态等用 per-item 布尔字段，别在 {{}} 里调方法。
微信 button 默认样式会顶掉宽度/居中——满宽用 width:100%!important，文字居中用 display:flex，并 ::after{border:none}。
onShareAppMessage 必须同步返回：需要 menuId 时先同步生成、后台建快照，不能 await。openid 在 app 启动异步预取，分享/接单依赖它。
头像昵称无法静默获取（微信 2022 政策）：只能用 open-type="chooseAvatar" + type="nickname" 一键授权。
